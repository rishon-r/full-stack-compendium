
- UPDATE statement:

The UPDATE statement is used to change the rows inside a table

General syntax:

UPDATE <tablename>
SET <col> = <val>
WHERE <condition>

If WHERE is not mentioned, it will change all the rows in the table

This will set value of given column to new value mentioned in all rows that satisfy that condition

E.g

UPDATE people
SET employed=true
WHERE JOB IS NOT NULL

You can also update multiple columns at once:

UPDATE people
SET employed = true, salary = 50000
WHERE job IS NOT NULL;

- DELETE statement:

The DELETE statement is used to remove all rows from a table that satisfy a following condition

In the most basic form it is:

DELETE FROM <tablename>

This will delete all rows from a table but won't delete the table itself

More commonly, you will see it used as follows:

DELETE FROM <tablename>
WHERE <condition>

E.g

DELETE FROM people
WHERE age > 60;

- The ALTER TABLE command

ALTER TABLE is a DDL (Data Definition Language) command used to modify the structure of an existing table — without touching the data inside it.

E.g ADDING A COLUMN

ALTER TABLE users
ADD COLUMN email TEXT;

With a default value:

ALTER TABLE users
ADD COLUMN active BOOLEAN DEFAULT true;

e.g DROPPING A COLUMN

ALTER TABLE users
DROP COLUMN email;

If you're not sure the column exists:

ALTER TABLE users
DROP COLUMN IF EXISTS email;

Renaming a column:

ALTER TABLE users
RENAME COLUMN name TO full_name;

Setting and dropping default values:

Set a default:
ALTER TABLE users
ALTER COLUMN active SET DEFAULT true;

Remove a default:
ALTER TABLE users
ALTER COLUMN active DROP DEFAULT;

Adding constraints:

ALTER TABLE users
ADD CONSTRAINT unique_email UNIQUE (email);

ALTER TABLE users
ADD CONSTRAINT users_pkey PRIMARY KEY (id);
