QUERYING tables

- We use SQL (structured query language) in order to select particular rows from a tablename
- THis process is known as querying and is done with the SELECT statement

TO get all rows from table: SELECT * FROM <tablename>

E.g SELECT * FROM people;

- WHERE clause

The WHERE clause when used in conjunction with the SELECT statement is used to filter the rows we query
It allows us to pick only those rows that adhere to some particular condition

E.g

SELECT name FROM people
WHERE age<40;

- General syntax of select statement that we will see:

SELECT <col1>, <col2>, ... , <colN>
FROM <tablename>
WHERE <condition>

- More on the condition:

The condition is usually a combination of a column name and some relational operator <, > , <=, >=, = , <>

Multiple conditions can be joined with the AND and OR operator

- The LIKE operator can also be used in the condition of where clause:

We say WHERE <col> LIKE 'pattern'

The LIKE operator uses two special character: _ meaning single character and  % meaning any number of characters

Example: WHERE name LIKE 'R%' will match any name that starts with R

WHERE name LIKE '_s%' will match any name with second letter s

- DISTINCT clause: DISTINCT is used to remove duplicate values from your results, returning only unique values

Syntax: SELECT DISTINCT column_name FROM table_name;

- BETWEEN operator: Used to select those values that belong in a range

Note that the between clause is inclusive on upper and lower limits

E.g 

SELECT name FROM people 
WHERE age BETWEEN 12 and 30

We can similarly use NOT BETWEEN

- Membership: You can also select values that only belong to a particular collection. We do this with the IN operator
E.g

SELECT name FROM people
WHERE job IN ('Electrician', 'PLUMBER');

We can similarly use NOT IN in order to achieve the opposite

- ORDER BY clause:

The order by clause allows you to order results based on a particular attribute

You can also order by multiple columns, in which case the columns are given a preference based on the order they are listed in

General syntax:

SELECT <col1>, .. , <colN>
FROM <tablename>
ORDER BY <col3>, <col4>, .., <colN>

Note that typically one doesn't order by more than one or two columns

SELECT name, age, city
FROM users
ORDER BY age DESC, name ASC;

Above example
Orders by age first (oldest to youngest)
If two people have the same age, orders them alphabetically

DESC means descending order, ASC means ascending
If ASC or DESC is not specified, postgres defaults to ASC

GROUP BY statement: 

The GROUP BY clause groups rows that have the same values in specified columns into summary rows.
It is typically used with aggregate functions like COUNT, SUM, AVG, MIN, MAX

General syntax:

SELECT <col>, <aggregate_function>(<col>)
FROM <tablename>
GROUP BY <col>

E.g: 

SELECT city, COUNT(*) 
FROM users
GROUP BY city;

It is illegal to have columns in the select statement that aren't mentioned in group by statement or aggregate functions of other columns

MORE EXAMPLE:

- COUNT: how many rows in each group
SELECT city, COUNT(*) 
FROM users 
GROUP BY city;

- SUM: total salary per department
SELECT department, SUM(salary) 
FROM employees
GROUP BY department;

- AVG: average salary per department
SELECT department, AVG(salary) 
FROM employees
GROUP BY department;

- MIN/MAX: youngest and oldest per city
SELECT city, MIN(age), MAX(age) 
FROM users
GROUP BY city;

You can also have multiple columns in the group by statement, example below:

SELECT city, job_title, COUNT(*)
FROM users
GROUP BY city, job_title;

Above example groups by unique combinations of city AND job title

- HAVING clause:

GROUP BY generally is used in combination with the HAVING clause

HAVING is used to filter groups, similar to how WHERE filters rows

The key difference:

WHERE filters rows before grouping
HAVING filters groups after grouping

General syntax:

SELECT <col>, <aggregate_function>(<col>)
FROM <tablename>
GROUP BY <col>
HAVING <condition>

Note: HAVING can only have conditions involving columns in the GROUP BY statement or AGGREGATE FUNCTIONS

E.g

SELECT city, COUNT(*) 
FROM users
GROUP BY city
HAVING COUNT(*) > 5;

Above example only shows cities that have more than 5 users

General order of clauses: 

SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY

E.g 

SELECT city, COUNT(*)
FROM users
WHERE age > 18
GROUP BY city
HAVING COUNT(*) > 5;

- COLUMN ALIASES:

A column alias is used to rename a column in your output using the AS keyword

E.g 

SELECT name, salary * 12 AS annual_salary
FROM employees;

Instead of showing ?column? or salary * 12 as the header, it will show annual_salary

General Syntax:

SELECT <col> AS <alias>
FROM <tablename>;
