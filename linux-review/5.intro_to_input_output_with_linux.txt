5. INPUT/OUTPUT
- Command 23: man command
    - man command is used to give us a manual that describes what a command
    does.
    - This is a multi page document that can be navigated.
    - We press ‘q’ on the keyboard to return back to the shell.
    - We pass the command we need to know about as the argument.
    - This provides a detailed description of the command along with all the flags
    that can be passed to it.
- Flags
    - Some commands accept flags
    - Flags are options you can pass to a command in order to change its
    behaviour
    - Example:
        - ls -l (l stands for long) which lists all files alongside their permissions
        - ls -a (a stands for all) lists all files including hidden files
    - You can also combine flags: ls -al
    - Whether or not a command takes flags depends on the command
    - Single character flags are prefixed with a single dash (E.g -a)
    - Multi character flags are prefixed with a double dash (E.g –help)
    - Sometimes the same flag can be used with a single dash or double dashed
    (E.g -h or –help)
- Positional Arguments
    - Interesting thing about flags is that the order in which they are listed does not
    matter
    - Programming languages such as python have functions that take arguments
    and can be called with those arguments.
    - In the shell, commands can also take arguments
        - E.g cd ~
    - Here, the alias for home directory is passed as an argument.
    - Other commands may take multiple arguments
        - Example: The mv command takes two arguments, the source file and
        destination file.
- Help
    - By convention most production ready CLI tools have a “help” option that prints
    what a command does.
    - This is usually accessed in one of the three ways:
        - --help flag passed along command
        - -h flag passed along with command
        - help as the first positional argument
    - The result of the help option is usually easier to read/parse than an entire
    manual page generated by man
- Exit Codes
    - Exit codes (also known as “return codes” or “status codes”) are the means by
    which programs communicate whether they ran successfully or not.
    - 0 is the exit code which means the program has run successfully
    - Any other exit code means the program did not run successfully
    - If a non zero exit code is returned, it will most probably be 1, which is the
    “catch all exit code”
    - Programs that call other programs use exit codes to see if the other program
    has run successfully or not
    - In shell, you can access the exit code of the last program you ran using the ?
    variable.
    - echo $? Will print out the exit code of the program you last ran.
- Standard Output
    - Standard output or stdout is the default place where programs print their
    output.
    - It is essentially just a stream of data that prints directly to your terminal
    - You can redirect this stream using more advanced techniques
    - All programming languages have a simple way to print to stdout
    - In Python, for example, it is the print() function
    - In the shell, it is the echo command
- Standard Error
    - Standard Error (stderr) is also just a data stream similar to stdout
    - However, it is intended for use when an error has occurred
    - It is a separate stream and can be redirected to a different location if need be
    - But, by default, it prints to the terminal just like stdout
- Stream redirection
    - You can redirect stdout and stderr to different places using the > and 2>
    operators respectively.
    - > redirects stdout, 2> redirects stderr
    - In practice usually, we redirect them to different files
    - E.g echo “redirected stdout” > a.txt
    - E.g cat doesnotexist.txt 2> error_msg.txt
    - You can then cat these files to see their contents.
- Standard Input
    - Standard Input is the default place where programs read their input.
    - It refers to a stream of data that programs can read from as they run.
    - All main programs provide a way to read from standard input.
    - For example, Python uses the standard input function
    - The shell uses the read command to read from the standard input
- Pipes
    - The | character is called the pipe character.
    - The pipe character takes the standard output of the program on the left of it
    and pipes it into the standard input of the program on the right of it.
    - Lot of commands work when you pipe in stdout as stdin such as grep
    - Use of grep with pipes serves as a very powerful tool
- Interrupt
    - Sometimes a program will get stuck and you will want to stop it
    - Common reasons for this are:
        - You made a typo in your command and it’s not doing what you want
        - It’s trying to access the internet but you are not connected
        - It’s processing too much data and you don’t want to wait for it to finish
        - There is a bug in the program causing it to hang
        - In all the cases mentioned above and more, you can stop the running of this
        program by pressing Ctrl + C
        - This sends a SIGINT signal to the program which tells it to stop
- Kill
    - Sometimes you’re in such a bad state that SIGINT doesn’t work, in which
    case the best option is to use another shell session (new terminal window) to
    manually kill the program.
    - Syntax: kill PID
    - Here, PID refers to process ID
    - Every process that runs on your computer has a unique PID.
- Command 24: ps command
    - Lists all processes running in your computer
    - ps stands for “Process status”
    - ps aux tells us to list all processes running on the computer including
    those owned by other users and lists more information about each
    process.
- UNIX Philosophy
    - The UNIX philosophy is a set of principles that guided the development of
    unix-like operating systems for decades. It can be summarized as follows:
    - Write programs that do one thing and do it well
    - Write programs that work together well
    - Write programs that handle text streams well
    - 1. Write programs that do one thing and do it well
        - Programs like ls, grep and less do one thing and do it well
        - They don’t try to do too much i.e go beyond the scope of what they
        are assigned to.
    - 2. Write programs that work together well
        - This is what makes piping such an effective tool
- Command 25: top command
    - top is a powerful command which allows you to see which programs are using
    the most resources in your computer
    - top is a more powerful analogue of the Windows Task Manager for the linux
    command line
    - top sorts the processes by CPU usage, with the most CPU-intensive
    processes at the top
    - Another useful metric to sort processes by is RAM usage or memory usage.
    To do this press M (uppercase) when top is running.
    - Exit the top command by pressing q
